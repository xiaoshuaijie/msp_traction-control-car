# MPU6050 与 NRF24L01 接入 app_main 实施计划

> **已废弃（2026-07-13）**：不得再执行本计划中的 NRF24L01 C API 接入步骤。
> 当前 Topic/Application 实现及验证要求以 `.helloagents/modules/nrf24l01.md` 为准。

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不改变现有车辆控制行为的前提下，让 `app_main.cpp` 周期驱动 MPU6050，并以 10 ms 周期接收 NRF24L01 数据及导出调试状态。

**Architecture:** MPU6050 通过工程已有的 `LibXR::MSPM0I2C` 适配器注册到 `HardwareContainer`，并由现有 `ApplicationManager::MonitorAll()` 驱动。NRF24L01 保持现有 C 接口和软件 SPI 驱动，在主循环中使用独立时间戳轮询；接入层只复制接收数据和记录状态，不控制电机。PowerShell 结构验证脚本覆盖构建入口和两个模块的关键接线，最终由 ARM 固件构建验证类型与链接关系。

**Tech Stack:** C++20、C11、LibXR、TI MSPM0 DriverLib/SysConfig、CMake、Ninja、PowerShell、GNU Arm Embedded Toolchain

---

在 MPU6050 状态同步之后、按键事件处理之前增加：

```cpp
if (now - last_nrf24l01_poll_time >= kNrf24l01PollPeriodMs)
{
  last_nrf24l01_poll_time = now;
  const uint8_t receive_status = NRF24L01_Receive();
  g_nrf24l01_receive_status = receive_status;
  if (receive_status == 1U)
  {
    std::memcpy(g_nrf24l01_rx_data, NRF24L01_RxPacket,
                NRF24L01_RX_PACKET_WIDTH);
    g_nrf24l01_receive_count++;
  }
  else if (receive_status == 2U || receive_status == 3U)
  {
    g_nrf24l01_error_count++;
  }
}
```

不在这里调用 `NRF24L01_Init()`：驱动在返回 `2` 或 `3` 前已经自行重新初始化。
