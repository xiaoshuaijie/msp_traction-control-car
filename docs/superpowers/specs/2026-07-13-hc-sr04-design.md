# HC-SR04 超声波模块设计

## 目标

在 `module/HC_SR04` 中实现一个由 `LibXR::ApplicationManager` 周期驱动的
HC-SR04 超声波模块。模块使用 MSPM0 PWM 定时器持续生成标准触发脉冲，使用
Timer Capture 测量 Echo 高电平宽度，并通过 LibXR Topic 发布带状态的距离样本。

本次工作同时修正现有 SysConfig 捕获模式和构建接入，但不在 `app_main.cpp` 中
实例化模块，避免把传感器开发扩大到整车业务逻辑。

## 范围

包含：

- 实现 `module/HC_SR04/sr04.h` 和 `module/HC_SR04/sr04.cpp`。
- 添加硬件无关的距离换算与滤波逻辑，供主机单元测试使用。
- 添加 `MODULE MANIFEST V2`、模块 CMake 配置和顶层 CMake 接入。
- 将 `CAPTURE_ULTRASONIC` 改为 Echo 高电平脉宽捕获，保持 1 MHz 计数时钟。
- 同步更新 SysConfig 源文件和当前已生成的 `ti_msp_dl_config.c/.h`。
- 添加距离换算、范围状态和滤波行为测试，并执行固件完整编译。

不包含：

- 修改 `app_main.cpp` 或整车控制策略。
- 避障、停车或速度规划逻辑。
- 温度补偿或多超声波传感器调度。
- 为 Timer Capture 新增 LibXR 通用驱动抽象。

## 方案选择

<a id="hc-sr04-d001"></a>
### hc_sr04#D001：使用 PWM Trigger 与 OnMonitor 轮询 Capture

采用 `OnMonitor()` 轮询 Timer Capture 原始事件标志的方案。硬件完成脉宽捕获，
应用回调只在出现新捕获事件时读取 Capture/Compare 寄存器并清除事件标志。

没有采用 Capture ISR，因为约 60 ms 的触发周期远低于 `MonitorAll()` 的常规轮询
频率，中断不会带来实际精度收益，却会增加全局中断入口和共享状态。也不直接周期
读取 Capture 寄存器，因为这种方式无法区分新回波和旧值。

## 公共接口

模块类命名为 `HC_SR04`，继承 `LibXR::Application`。

### Resources

`Resources` 描述模块所需硬件资源：

- `LibXR::MSPM0PWM::Resources trigger_pwm`：Trigger PWM 定时器、通道和源时钟。
- `GPTIMER_Regs* capture_timer`：Echo Capture Timer。
- `DL_TIMER_CC_INDEX capture_channel`：Echo 捕获通道。
- `uint32_t capture_event_mask`：对应捕获通道的原始事件标志。
- `uint32_t capture_clock_hz`：Capture Timer 实际计数频率，默认接入为 1 MHz。

资源由接入层传入，模块不硬编码 TIMG7/TIMG8，也不依赖当前不完整的
`MSPM0_PWM_INIT` 宏。

### Config

`Config` 提供以下默认值：

- Topic 名：`hc_sr04`
- Trigger 频率：`16 Hz`，实际周期约 `62.5 ms`
- Trigger 高电平宽度：`10 us`
- 有效距离下限：`20 mm`
- 有效距离上限：`4000 mm`
- EMA 旧值权重：`0.6`
- 无回波超时：`100 ms`

构造阶段校验资源和配置，配置 PWM 频率与占空比，清除旧捕获标志，启动 Capture
Timer 和 Trigger PWM，最后注册到 `ApplicationManager`。

### Sample

Topic 发布的 `Sample` 包含：

- `uint32_t pulse_width_us`：Echo 高电平宽度。
- `float raw_distance_mm`：当前脉宽换算出的原始距离。
- `float filtered_distance_mm`：最近有效距离经过 EMA 后的结果。
- `Status status`：`VALID`、`NO_ECHO`、`BELOW_MIN` 或 `ABOVE_MAX`。
- `uint32_t sequence`：每次 Topic 发布递增的序号。

公共查询接口包括 `SampleTopic()`、`GetLatestSample()`、`HasValidSample()` 和
`OnMonitor()`。订阅者必须根据 `status` 判断当前帧是否可用于控制；无效帧中的
`filtered_distance_mm` 仅表示最近一次有效滤波值。

## 数据流

```text
MSPM0PWM (Trigger, 16 Hz / 10 us)
  -> HC-SR04 Trigger
  -> HC-SR04 Echo
  -> Timer Capture (1 MHz pulse-width capture)
  -> OnMonitor() 检查原始捕获事件
  -> 脉宽换算、范围判断、EMA
  -> Topic<HC_SR04::Sample>("hc_sr04")
```

脉宽换算公式为：

```text
pulse_width_us = capture_ticks * 1,000,000 / capture_clock_hz
raw_distance_mm = pulse_width_us * 0.17
```

`0.17 mm/us` 表示按约 `340 m/s` 声速计算单程距离。

## OnMonitor 状态机

每次 `OnMonitor()` 按以下顺序执行：

- 读取 Capture 原始事件标志。
- 若存在新事件，读取脉宽计数并清除事件标志。
- 将计数换算为微秒和毫米，并分类为有效、过近或过远。
- 有效帧更新滤波器：首帧直接使用原始距离，后续帧使用
  `filtered = old * 0.6 + raw * 0.4`。
- 过近或过远帧照常发布状态，但不更新滤波器。
- 若超过无回波超时仍没有新捕获，则按 Trigger 周期发布 `NO_ECHO`；脉宽和原始
  距离置零，滤波距离保持最近一次有效值。
- 每次发布前递增序号并更新最近样本。

所有时间差使用无符号减法，保证系统毫秒计数回绕时仍能正确工作。Capture 原始
事件在读取后清除，防止重复发布同一寄存器值。

## SysConfig

`CAPTURE_ULTRASONIC` 从上升沿周期捕获改为上升沿开始的高电平脉宽捕获，使用向上
计数模式。Capture Timer 保持 1 MHz，使一个计数对应 `1 us`，50 ms 周期足以覆盖
HC-SR04 的有效 Echo 脉宽。

PWM 的最终频率和占空比由 `MSPM0PWM` 在模块构造阶段设置；SysConfig 继续负责
Timer 和引脚复用的基础初始化。SysConfig 源文件和当前生成文件必须保持一致。

## 错误处理

- 空 Timer 指针、零时钟、零频率、非法距离范围或超出 `[0, 1]` 的滤波权重通过
  `ASSERT` 阻止启动。
- `MSPM0PWM::SetConfig()`、`SetDutyCycle()`、`Enable()` 失败时通过 `ASSERT`
  阻止模块以不完整配置运行。
- 无回波、过近和过远属于运行状态，通过 Topic 发布，不作为程序错误。
- 无效测量不会覆盖最近一次有效滤波值，也不会把固定哨兵值送入滤波器。

## 测试与验收

硬件无关逻辑通过主机测试验证：

- 1 MHz Capture 下 `1000 us` 换算为约 `170 mm`。
- 非 1 MHz Capture 时换算仍按资源时钟正确缩放。
- 第一帧有效测量直接初始化滤波值。
- 后续有效测量按 `0.6/0.4` 权重更新。
- 过近、过远和无回波返回正确状态，且不改变滤波值。
- 有效边界值 `20 mm` 和 `4000 mm` 可被接受。

集成验收包括：

- SysConfig 生成配置使用 `DL_TIMER_CAPTURE_MODE_PULSE_WIDTH_UP`。
- Topic 类型、模块 Manifest 和 CMake 接入一致。
- 主机单元测试通过。
- ARM 固件 CMake 配置与完整构建通过。
- 不修改或提交工作区中与本模块无关的现有改动。
